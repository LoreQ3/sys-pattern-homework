### Задание 1. Кеширование 

Приведите примеры проблем, которые может решить кеширование. 

1. Высокая задержкаи при доступе к данным. 
Кеширование снижает время отклика: данные, хранящиеся в кеше, возвращаются в сотни раз быстрее.
Примеры: Веб-сайты, базы данных, операционные системы.
2. Низкая пропускная способность
Основной источник данных не способен обслуживать большое количество одновременных запросов.
Кеширование поглощает пиковые нагрузки: кеш выступает как буфер, принимая на себя основной поток чтения.
Примеры: Новостные сайты во время крупных событий, API популярных сервисов.
3. Чрезмерная нагрузка на основные ресурсы
Постоянные запросы к основному источнику данных (БД, микросервис) приводят к его перегрузке, увеличению времени ответа и даже отказу в обслуживании.
4. Избыточные вычисления
Одно и то же сложное вычисление выполняется снова и снова для каждого запроса, хотя его результат не меняется.
Кеширование сохраняет результат вычисления: вычисление выполняется один раз, а его результат сохраняется.
Примеры: Генерация миниатюр изображений, рендеринг сложных страниц, аналитика и отчеты
5. Согласованность данных в распределенных системах
В сложных системах (например, микросервисная архитектура) разные сервисы могут иметь свои собственные копии данных для независимой работы.
Что решает кеширование (в данном контексте — шаблон "Кеш по стороне"): позволяет сервису работать автономно: сервис хранит локальную копию (снепшот) данных, 
которые ему нужны от других сервисов.
Сервис может продолжать работу, даже если сервис-источник данных временно недоступен.
Пример: Сервис заказов может хранить в своем локальном кеше копию каталога товаров (названия, цены) из сервиса каталога.
